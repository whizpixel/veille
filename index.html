01 — Présentation

C'est quoi Veille Tech Bot ?

Veille Tech Bot est un bot Discord développé dans le cadre du BTS SIO. Il automatise entièrement le processus de veille informatique sur l'intelligence artificielle, une tâche jusqu'alors réalisée manuellement.

Avant ce bot, la veille nécessitait de consulter Feedly, copier chaque article dans ChatGPT pour en obtenir un résumé, puis saisir manuellement les données dans un tableur. Désormais, tout est automatique.

02 — Fonctionnement

Comment ça marche ?

Le bot suit un processus en 4 étapes, exécuté automatiquement chaque jour à 7h du matin.

1
Récupération des flux RSS

Le bot lit les flux RSS configurés et récupère les derniers articles. Les articles déjà traités sont ignorés grâce à un système anti-doublons.

2
Génération du résumé via Gemini

Le contenu de chaque article est envoyé à l'API Google Gemini 2.5 Flash, qui génère un résumé de 3 lignes en français.

3
Publication sur Discord

Un message formaté est posté dans le canal Discord configuré, avec le titre cliquable, le résumé et la source.

4
Export vers Google Sheets

Les données (date, titre, lien, résumé, source) sont automatiquement ajoutées dans le tableur Google Sheets.

Architecture technique

Le bot est hébergé sur un mini PC sous Ubuntu Server, connecté en permanence à internet. Il fonctionne en tant que service systemd, ce qui lui permet de démarrer automatiquement au reboot.

03 — Utilisation

Guide utilisateur

Le bot fonctionne de manière entièrement autonome. Voici comment interagir avec lui.

Lire les articles sur Discord

Chaque matin à 7h, le bot poste automatiquement les nouveaux articles dans le canal Discord. Chaque message contient :

📰 [Titre de l'article] (cliquable vers l'article original)
Le résumé généré par Gemini en 3 lignes.
🔗 Source : Nom du site

Ajouter un flux RSS

Pour ajouter une nouvelle source, modifie la liste RSS_FEEDS dans bot.py :

bot.py
RSS_FEEDS = [
    "https://www.lemonde.fr/intelligence-artificielle/rss_full.xml",
    "https://nouvelle-source.com/rss.xml",  # Ajouter ici
]
⚠️ Après modification, redémarre le service : sudo systemctl restart discord-bot

Gérer le service

Terminal SSH
# État du bot
sudo systemctl status discord-bot

# Redémarrer
sudo systemctl restart discord-bot

# Arrêter
sudo systemctl stop discord-bot

# Voir les logs en temps réel
sudo journalctl -u discord-bot -f
✅ Le bot redémarre automatiquement au reboot du serveur grâce à systemd.

04 — FAQ

Questions fréquentes

Pourquoi le résumé affiche "indisponible" ?

Le quota gratuit de l'API Google Gemini est limité à 20 requêtes par jour. Si cette limite est atteinte, le bot poste l'article sans résumé. Le quota se renouvelle automatiquement le lendemain.

Le bot peut-il poster deux fois le même article ?

Non. Le bot conserve un fichier posted_articles.txt contenant les liens déjà traités. Un article déjà posté sera ignoré lors des prochaines exécutions.

À quelle fréquence le bot s'exécute-t-il ?

Le bot s'exécute une fois par jour à 7h00, heure locale du serveur.

Comment accéder au serveur ?

Le serveur est accessible via SSH depuis le réseau local :

ssh yunus@192.168.1.135
